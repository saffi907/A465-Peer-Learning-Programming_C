What Went Wrong (and How to Fix It)

Stage 1 Info Leak
Here's what went wrong: A debug API endpoint (/api/debug) was left in the code, leaking secret keys, internal routes, and stuff that shouldn't be public.
How to stop it: Remove debug endpoints before deploying. Store secrets in environment variables, not in code.
Stage 2 Mass Assignment
Here's what went wrong: The registration form accepted all fields you threw at it, so you could sneak in role=admin and the server would just go with it.
How to stop it: Only grab the fields you actually expect (username, password). Ignore everything else.
Stage 3 Template Injection
Here's what went wrong: User input went straight into render_template_string(), so any Jinja2 code you typed would just run on the server and leak config secrets.
How to stop it: Don't pass raw user input to the template engine. Use render_template() with actual template files and escape everything.
Stage 4 Path Traversal
Here's what went wrong: The file browser stuck user input directly into a file path with no sanitization, so ../ let you escape the folder and read files you shouldn't have access to.
How to stop it: Use os.path.basename() to strip directory parts. Or use an allowlist of files that are okay to serve.
